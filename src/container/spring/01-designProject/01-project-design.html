프로젝트 요구사항

As a [user role], I want [goal], so that [rationale].

  1. Requirements(요구사항 분석) : user stories
    • Artifact Management
      • Anonymous user : find all artifact, find artifact by id
      • logged-in user : add artifact, update artifact, delete artifact
    • Wizard Management
      • logged-in user: find all wizard, add wizard, find wizard by id, update wizard, delete wizard, assign artifact to wizard
    • User Management
      • admin user : find all user, add user, find user by id, update user, delete user, login
  2. Planning(어찌할지 계획 수립): GitHub Issues
  3. Design(설계 하기) : API-first Approach, design a user story
  4. implementation and Testing : TDD
  5. Deployment : CI/CD
